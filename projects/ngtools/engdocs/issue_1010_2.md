### Task

`agent-prototypes/agent_prototypes/configs/experiment/qabench_test.yaml` should support a repo_pattern field.
Only repos matching provided pattern will be used in `agent-prototypes/agent_prototypes/scripts/run_question_generation.py`

### analysis

The script `agent-prototypes/agent_prototypes/scripts/run_question_generation.py` loads all repository instances from the dataset specified in the experiment config. It does not have a mechanism to filter repositories based on a pattern.

### fix_plan

1.  Modify `agent-prototypes/agent_prototypes/scripts/run_question_generation.py` to read an optional `repo_pattern` from the experiment configuration file.
2.  If `repo_pattern` is present, use the `re` module to filter the loaded `QABenchInstance` objects, keeping only those where the `repo` attribute matches the pattern.
3.  Add a `repo_pattern` field to `agent-prototypes/agent_prototypes/configs/experiment/qabench_test.yaml` with a default value of `.*` to match all repositories.
