### Task

questions in the pipeline contains an origin field

```
  {
    "category": "Process",
    "question": "What's the intended workflow for adding a new CI check that should only run on the main branch, to avoid having to manually modify the YAML files in release branches later on?",
    "question_id": "94ca6da3",
    "origin": "issues"
  },
```

This is not included in verification synthezied and later golden sets.
It should be.

The assumptions is that all questions used for synthesis will have the same origin for now.


### analysis

The `origin` field is being dropped at two key stages in the QABench pipeline:

1.  **Answer Synthesis (`qabench_synthesizer.py`)**: The synthesizer script constructs its `question_data` from the `question` object within the individual `_answer.json` files. This `question` object is a subset of the original question and does not include the `origin` field. As a result, the output `_synthesized_answer.json` file lacks this field.
2.  **Golden Set Filtering (`qabench_answerer.py filter`)**: The `filter` command reads the original `_questions.json` file to get the question text and category, but it does not extract the `origin` field when creating the final `golden_entry` for the `golden_answers.jsonl` file.

### fix_plan

1.  **Modify `qabench_synthesizer.py`**: Update the script to load the main `INSTANCE_ID_questions.json` file. It will use the `question_id` from the answer files to look up the complete original question data, including the `origin` field, and include this complete data in the `_synthesized_answer.json` output.
2.  **Modify `qabench_answerer.py`**: Update the `run_filter` function to explicitly read the `origin` field from the `original_question_data` and add it to the `golden_entry` dictionary.
