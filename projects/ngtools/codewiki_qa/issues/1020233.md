### Task

when running the qa bench pipeline agent-prototypes/agent_prototypes/scripts/qabench_pipeline.py
I need to collect metrics on gemini cli token usage.

I know behind the scene, all gemini cli usage are routed to the gemini cli runner, is there a way to collect the token usage metrics?

### analysis

The user wants to collect token usage metrics from the Gemini CLI when running the `qabench_pipeline.py` script.

The investigation of the codebase reveals the following:

1.  The `qabench_pipeline.py` script orchestrates calls to other scripts, primarily `qabench_answerer.py`.
2.  `qabench_answerer.py` uses a utility function `run_gemini_cli_async` from `agent_prototypes/utils/gemini_cli_runner.py` to execute Gemini CLI commands.
3.  The `run_gemini_cli_async` function in `gemini_cli_runner.py` has a parameter `telemetry_log_file`. If a file path is provided for this parameter, the function adds the `--telemetry`, `--telemetry-target=local`, and `--telemetry-outfile` flags to the `gemini` CLI command.
4.  This telemetry feature is the mechanism to collect token usage and other metrics from the Gemini CLI.

The root cause of the missing metrics is that the telemetry feature is not currently being used in the `qabench` pipeline.

### fix_plan

To enable token usage metrics collection, I will make the following changes:

1.  **`agent_prototypes/utils/gemini_cli_runner.py`**: No changes are needed in this file as the telemetry functionality is already implemented.

2.  **`agent_prototypes/scripts/qabench_answerer.py`**:
    *   Update the `run_gemini_cli_async` calls in `QABenchAnswerer.answer_question` and `QABenchVerifier.verify_answer` to pass the `telemetry_log_file` parameter.
    *   Add a new `--telemetry-log-dir` command-line option to the `cli_answer` and `cli_verify` functions to accept a directory path for storing telemetry logs.
    *   Inside the `answer_question` and `verify_answer` methods, if the `telemetry_log_dir` is provided, a unique file path for each telemetry log will be constructed and passed to `run_gemini_cli_async`.

3.  **`agent_prototypes/scripts/qabench_pipeline.py`**:
    *   In the `process_single_question` function, read a new `telemetry_log_dir` setting from the YAML config file.
    *   If this setting is present, add the `--telemetry-log-dir` argument to the command-line calls for `qabench_answerer.py`.

4.  **YAML Configuration**:
    *   The user will need to add a `telemetry_log_dir` key to their YAML configuration file, specifying the directory where telemetry logs should be saved. For example:
        ```yaml
        experiment_dir: experiments/qabench/your_experiment
        telemetry_log_dir: experiments/qabench/your_experiment/telemetry_logs
        concurrency: 2
        num_answers: 3
        ```

This plan will enable the collection of token usage metrics in a structured way, without modifying the core Gemini CLI runner.
