### analysis

The user wants to aggregate token usage data from a directory of telemetry logs generated by the Gemini CLI. Each log file is a JSON object containing various metrics, including the desired token counts (`input_token_count`, `output_token_count`, etc.) within the `attributes` field. Currently, there is no tool to automate the extraction and aggregation of this data. A script is needed to parse these log files, sum up the relevant token metrics, and output a summary.

### fix_plan

1.  **Create a new Python script** named `aggregate_telemetry.py` inside the `agent-prototypes/agent_prototypes/scripts/` directory.
2.  The script will accept a single command-line argument: the path to the telemetry directory.
3.  The script will iterate through all files in the specified directory.
4.  For each file, it will:
    a. Read the content and parse it as JSON.
    b. Check if the `attributes` key exists and the `event.name` is `gemini_cli.api_response`.
    c. Extract the values for `input_token_count`, `output_token_count`, `cached_content_token_count`, `thoughts_token_count`, `tool_token_count`, and `total_token_count`.
    d. Add the extracted values to a running total for each metric.
5.  After processing all files, the script will:
    a. Print the aggregated totals for each metric to the console.
    b. Write the aggregated totals to a new file named `token_usage_summary.json` in the input telemetry directory.
