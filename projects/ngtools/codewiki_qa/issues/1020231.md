### analysis

The `agent-prototypes/agent_prototypes/qabench/repos.csv` file contains duplicate repository entries. Some of these duplicates point to different commits of the same repository. This leads to incorrect data in `codewiki_qa/curation/experiment_2` because the same repository is being considered multiple times, potentially with inconsistent states. The process that generates `repos.csv` should be reviewed to ensure each repository is listed only once with its correct commit.

### fix_plan

1.  Create a Python script to read `agent-prototypes/agent_prototypes/qabench/repos.csv`.
2.  The script will process the CSV to remove duplicate repository entries, keeping only the most recent commit for each unique repository.
3.  The script will overwrite `agent-prototypes/agent_prototypes/qabench/repos.csv` with the cleaned, unique repository data.
4.  A new curation experiment, `experiment_3`, will be set up under `codewiki_qa/curation/`.
5.  This new experiment will be configured to use the updated `repos.csv` file as its input.
6.  Run the curation process for `experiment_3` to generate a correct set of data based on unique repositories.
