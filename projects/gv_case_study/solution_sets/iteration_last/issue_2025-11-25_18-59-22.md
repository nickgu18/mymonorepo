### analysis
The core pipeline is implemented and tested, but the repository still reflects iterative notebook-driven development rather than a clean, review-ready package:

- `xgboost/src` contains helpers and modules that were useful during earlier iterations but are no longer referenced by tests or the final notebook, which adds noise and makes it harder for a reviewer to see the true surface area of the solution.
- `GV_Exercise_Iteration5.ipynb` mixes exploratory cells, long functions, and verbose comments with the final pipeline calls, so it is harder to scan as a concise narrative of the approach.
- `configs/base.yaml` hard-codes the MLflow tracking URI with an absolute path and still exposes `outputs` and `logging.redact_fields` fields that are either unused or only relevant for local experimentation, which hurts portability and over-complicates the config contract.
- There is no single command to bootstrap a virtual environment and install the project in editable mode, so reviewers must manually follow the README steps, increasing friction and the chance of setup errors.

### fix_plan
1. **Prune unused code under `xgboost/src`**
   - Treat `xgboost/tests` and the final notebook as the canonical consumers of the library.
   - Use import/search tooling (e.g. `rg`, simple static greps) to identify modules, functions, and types in `src/common`, `src/data`, `src/features`, `src/models`, and `src/predict` that are no longer imported anywhere.
   - Remove dead modules and functions, simplifying `__init__.py` files to export only the small set of public contracts actually used by tests / notebook, and re-run tests to confirm nothing breaks.

   - Strip out exploratory/debug cells, leaving a linear story: config load → data load/clean → feature building → training → evaluation → batch scoring.

   - Tighten comments and markdown so they are short, high-signal explanations of intent rather than step-by-step narration.

3. **Make `configs/base.yaml` portable and minimal**
   - Change `tracking.uri` to be expressed relative to `project.root` instead of embedding the current absolute path (for example, `file://${project.root}/mlruns`), relying on the existing placeholder resolution in `src/common/config.py`.
   - Remove the `outputs` section and the `logging.redact_fields` key from `base.yaml`, and then align the `Config` dataclasses and `_materialize` function in `src/common/config.py` so they no longer require or emit those fields.

4. **Add a simple `Makefile` for environment setup**
   - In `xgboost/Makefile`, add targets such as `venv`, `install`, and `test` that (a) create `.venv`, (b) install the project with `pip install -e .[dev]`, and (c) run `pytest`.
