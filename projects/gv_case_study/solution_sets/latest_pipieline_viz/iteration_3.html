Founder Ranking Pipeline

Iteration: 3

High-level Flow

Phase 1: Environment Setup
Configuration and initialization
  • Load configuration via load_config().
  • Set up MLflow tracking and logging.
  • Initialize random seeds for reproducibility.
↓
Phase 2: Data Loading & Feature Engineering
Load data and build founder features
  • 2.1 Data Loading & Cleaning:
    • Call load_raw for training and ranking datasets.
    • Call clean() to get RankingCleanData.
    • Call load_targets() for curated targets + founding metadata.
  • 2.5 Feature Matrix Construction:
    • Build FeaturePipeline founder matrix with education, experience, and company features.
    • Build label metadata using _build_last_labeled_company_metadata:
    • Filter founder rows with non-null company_id.
    • Inner-join with targets to keep only labeled companies.
    • Select the last labeled company per founder.
    • Join founder features with label metadata via build_training_dataset.
    • Compute integer labels via calculate_relevance_grade(multiple).
    • Remove train/inference leakage.
    • Result: 4,763 founders with labels and valid founding years.
  • 2.7 Manual Feature Engineering: Additional derived features if needed.
↓
Phase 3: Data Splitting
temporal_train_val_test_split_by_industry
  • Group training_df by industry.
  • Within each industry:
    • Sort by company_founded, company_id, person_id.
    • Split into ~70% Train, 15% Validation, 15% Test (adjusting small cohorts).
  • Return: X_train, y_train, X_val, y_val, X_test, y_test + groups.
↓
Phase 4: Data Validation
Verify split quality and data integrity
  • Print per-industry counts and label distributions for all three splits.
  • Verify temporal ordering (no train rows after test rows).
  • Check for missing values and feature ranges.
↓
Phase 5: Model Training
train_ranker with validation as eval set
  • Train XGBRanker with:
    • X_train, y_train, train_groups
    • X_val, y_val, val_groups as the eval set
  • MLflow logs parameters, label histograms, feature importance, and NDCG@20 for Train and Validation.
  • Validation is used for model selection / tuning; Test remains untouched.
  • After training, compute final Test NDCG@20 using calculate_mean_ndcg.
↓
Phase 6: Model Persistence
Save model and metadata
  • Persist the trained model and feature metadata under mlruns/models/notebook_runs/<timestamp>.
  • Save model bundle for later loading.
↓
Phase 7: Inference
Run predictions on ranking dataset
  • Reload the bundle with load_model_bundle.
  • Build ranking features via FeaturePipeline on clean_ranking.
  • Run predict_batch to get scores and rankings.
↓
Phase 8: Ranking Explanation
SHAP analysis for interpretability
  • Generate global and per-industry SHAP plots.
  • Compute top-N SHAP explanations for feature importance.
  • Visualize how different features contribute to founder rankings.
Iteration 3 keeps the shared library structure from Iteration 2 but fixes label attachment, removes unnecessary row loss, and moves to a per-industry temporal Train/Validation/Test split. Model tuning now uses the validation slice, and the test slice is reserved strictly for final reporting.
