Founder Ranking Pipeline

Iteration: initial

High-level Flow

1. Data Load
Notebooks + raw CSVs
  • Load founder_experience_training.csv, education_training.csv, company_info_training.csv.
  • Load target_variable_training.csv (company-level multiples).
  • Load separate ranking (inference) tables.
↓
2. Cleaning
Notebook logic
  • Normalize IDs and dates inside the notebook.
  • Basic filtering on founders and companies.
  • No shared library for cleaning yet; logic lives in analysis cells.
↓
3. Feature Matrix (Baseline)
Notebook-only feature building
  • For each founder, filter rows with is_founder = True.
  • Select the last founded company by order.
  • Join education and company_info for that company.
  • Aggregate performance features (mean / max / last) over founded companies.
  • Result: founder matrix with ~4,772 founders.
↓
4. Label Construction (Baseline)
Notebook joins
  • Left-join founder matrix with target_variable_training on company_id.
  • Compute label as floor(log1p(multiple)) for the last founded company.
  • Rows where the last company has no target row keep company_founded as NaN.
↓
5. Split & Drop
Per-industry temporal split
  • Split by industry and company_founded, older → Train, newer → Test.
  • Rows with missing company_founded (~25%) are dropped at this step.
  • No separate validation set; only Train/Test.
↓
6. Model Training & Inference
Notebook training
  • Train XGBoost ranker per industry using NDCG@K as objective/metric.
  • Evaluate on the single test split.
  • Run inference on ranking founders using the same feature logic.
This initial iteration implemented the core idea (founder-level ranking with multiples) entirely in notebooks, with a single per-industry Train/Test split and significant row loss due to missing company_founded after attaching labels to the last founded company.
