Click into each iteration to see how the data pipeline evolves. Or compare two iterations side-by-side.

Iterations

Initial Iteration
Notebook-only implementation: ad-hoc loading/cleaning, founder matrix built directly in the notebook, labels joined on the last founded company, and a single per-industry Train/Test split.
Iteration 2
Pipeline moved into shared library code (config, loaders, FeaturePipeline, training dataset, splitters, train_ranker) with a per-industry temporal Train/Test split and NDCG reporting.
Iteration 3
Refined label construction (last labeled company per founder), reduced row loss, and introduced a per-industry temporal Train / Validation / Test split, with training on Train+Val and true held-out evaluation on Test plus SHAP explanations.
Iteration 4 (Proposed)
Fixes temporal logic by using best exit (max multiple) company as the label instead of last labeled company. Features are strictly built from information available as of the founding date of that best company, and data-dependent transformations are fit on train only to prevent leakage.
