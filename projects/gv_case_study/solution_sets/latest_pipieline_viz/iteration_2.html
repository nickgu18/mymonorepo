Founder Ranking Pipeline

Iteration: 2

High-level Flow

1. Config & Data Load
Config-driven loading via src.common / src.data
  • Use load_config() to read configs/base.yaml.
  • Call load_raw(..., dataset="training") and load_raw(..., dataset="ranking").
  • Call load_targets() for target_variable_training.csv.
↓
2. Cleaning
src.data.loaders.clean
  • Normalize IDs (strip /p/, /c/ prefixes).
  • Parse dates in experience (start_date, end_date).
  • Coerce company_info.performance to numeric, preserving NaN.
  • Return RankingCleanData for training and ranking datasets.
↓
3. Founder Feature Matrix
FeaturePipeline.build_matrix
  • For each founder, build history from clean_training.experience.
  • Select the last founded company (by order, ignoring missing company_id).
  • Join education features (education_tier, highest_level → score).
  • Attach company context (industry, performance).
  • Aggregate performance across all founded companies: founder_has_perf, founder_perf_mean, founder_perf_max, founder_perf_last.
  • Emit one row per founder (4,772 rows) → FeatureMatrix.
↓
4. Label Construction & Leakage Removal
build_training_dataset
  • Join founder matrix with target_variable_training on company_id.
  • Attach company_founded, multiple, and target industry metadata.
  • Compute integer label via calculate_relevance_grade(multiple).
  • Remove founders whose last founded company appears as the last company in the ranking dataset (no train/inference leakage).
  • Coerce company_founded to numeric; rows with missing values were still being dropped later in the splitter.
  • Result: 4,769 founders with labels.
↓
5. Split: Train / Test (Per Industry)
split_by_industry
  • Drop rows with missing company_founded inside the splitter.
  • Group by industry; within each group, sort by company_founded, then split 80% Train / 20% Test.
  • Return:
    • X_train, y_train, X_test, y_test
    • train_groups, test_groups, industry_names
  • No separate validation set; Test was used to reason about underfitting vs. generalization.
↓
6. Model Training & Evaluation
train_ranker + NDCG
  • Train XGBRanker with: eval_set=[(X_train, y_train), (X_test, y_test)] and per-industry group arrays.
  • Log metrics and artifacts via MLflow.
  • Compute NDCG@20 per industry and macro Train/Test NDCG.
  • Test NDCG often > Train NDCG due to easier, label-0-heavy test cohorts.
Iteration 2 moved the pipeline into shared library code (config, loaders, FeaturePipeline, training_dataset, splitters, training) but still used a single Train/Test temporal split. This exposed label distribution shifts and underfitting, motivating the Train/Validation/Test design in iteration 3.
