Founder Ranking ML Pipeline

A visual walkthrough of how raw CSV files become ranked founder scores using a learning-to-rank model.

CSV files → Load data → Validate schema → Clean & join tables → Engineer features → Create labels (0–5) → Train ranker → Evaluate & save model → Score founders → Sort & display ranking
Data Labels Model Results
1
Raw CSV → dataframes
Data
You start with CSV files for founders, companies, and historical outcomes. They are loaded into dataframes and checked against a schema so that IDs, types, and required columns are consistent.
  • Read CSVs: founders, companies, exits, performance.
  • Validate: column names, data types, primary keys, null constraints.
  • Fail fast if something is missing or malformed.
csv → pandas.DataFrame → schema.validate(df)
2
Cleaning & joining tables
Data
Next you clean and align tables so that every row refers to the same entities, and IDs line up across files.
  • Normalize IDs (company_id, founder_id).
  • Drop impossible or inconsistent rows.
  • Join founders ↔ companies ↔ outcomes into a single training table.
companies ⨝ founders ⨝ outcomes → clean_training_table
3
Feature engineering
Data → Model
From each founder/company, you build numeric and categorical features that summarize their history and context.
  • Counts of past exits: 1–3x, 3–10x, 10x+.
  • Education tiers, markets, years of experience.
  • Aggregations over portfolios (mean performance, max MOIC).
raw_columns → engineered_features_matrix (X)
4
Label creation (target variable)
Outcome → Label
For past companies you know the outcome (for example, MOIC). You compress that continuous outcome into a discrete relevance label from 0 to 5.
  • Use historical MOIC / performance as ground truth.
  • Apply a function: outcome → grade ∈ {0…5}.
  • This grade is the training label (y), not the final founder score.
y = calculate_relevance_grade(moic)
5
Training the ranking model
Model
You feed features (X) and labels (y) into a learning-to-rank model such as XGBoost's XGBRanker. It learns to assign higher scores to rows with higher labels.
  • Objective: learn scores so that higher-label founders outrank lower-label ones.
  • Optimization uses many pairwise comparisons between rows inside each training group.
  • Output: a trained model that maps features → score.
ranker.fit(X_train, y_train, group=groups)
6
Evaluating & saving the model
Model
On validation data, you check whether high-label founders tend to receive higher scores, and you store the best model.
  • Compute ranking metrics (e.g. NDCG) to see how well the ordering matches labels.
  • Inspect feature importance / SHAP explanations for sanity.
  • Serialize the trained model artifact for later use in scoring.
metric = ndcg(y_valid, scores_valid) save_model(ranker, "founder_ranking_model.xgb")
7
Scoring new founders
Serving
For new founders, you repeat the data and feature steps, but there is no label yet. You simply ask the model for a score.
  • Build the same feature columns as during training.
  • Load the saved model and apply it: X_new → score.
  • Each founder gets a continuous score (e.g. 2.73).
scores = ranker.predict(X_new)
8
Ranking & presenting results
Output
Finally you sort founders by score and display them in order, optionally with explanations for transparency.
  • Sort descending by score → rank 1, 2, 3…
  • Attach per-founder explanations (key features that moved the score).
  • Render a clean table or UI view for decision-makers.
results = founders.assign(score=scores) results = results.sort_values("score", ascending=False)
