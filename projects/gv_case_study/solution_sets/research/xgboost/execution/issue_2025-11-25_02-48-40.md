### analysis

The loaded `ModelBundle` carries the **full training feature set** in
`feature_columns`, which includes the post‑split aggregates:
`founder_has_perf`, `founder_perf_mean`, `founder_perf_max`,
`founder_perf_last`, `network_quality_1st`, `network_size_1st`,
`team_size`.

During training (Phase 5/6 in
`xgboost/notebooks/GV_Exercise_Iteration4.ipynb`), these columns are
added on top of the base features by:

- computing founder‑level performance aggregates with
  `compute_founder_performance_aggregates`,
- computing network features with `compute_network_features`, and
- computing team size per label company with `compute_team_size`,
  then merging the results into `train_df` / `val_df` / `test_df`
  before fitting the model and persisting `artifacts.json`.

However, in Phase 7 the ranking path builds `ranking_features` via
`FeaturePipeline.build_matrix(clean_ranking)` (see
`xgboost/src/features/pipeline.py`). That pipeline currently only emits
the **base local features** from `cfg.features.selected_columns`
(`performance`, `education_tier`, `education_level_score`) plus a few
ID/metadata columns, and it does **not** call the post‑split helpers in
`feature_factory` or attach the derived columns that were present when
the model was trained.

As a result:

- `bundle.feature_columns` expects the extended feature set
  (base + performance/network/team columns),
- `ranking_features.frame` only contains the base feature columns,
- `FeatureMatrix.select_columns(bundle.feature_columns)` detects that
  those extra columns are missing and raises the `KeyError`.

So the root cause is a **feature mismatch between training and
inference**: the inference feature builder has not been updated to
compute / attach the post‑split aggregates that the trained model
expects, so the model’s recorded `feature_columns` and the ranking
feature matrix are out of sync.

1. **Confirm the expected feature set from artifacts**
   - Inspect the active `artifacts.json` under
     `xgboost/mlruns/models` (or the run‑specific directory used in
     Phase 6) and verify that `feature_columns` contains:
     `performance`, `education_tier`, `education_level_score`,
     `founder_has_perf`, `founder_perf_mean`, `founder_perf_max`,
     `founder_perf_last`, `network_quality_1st`, `network_size_1st`,
     `team_size`.
   - Add a short note in the notebook confirming that Phase 6 and
     Phase 7 are using the same artifact.

2. **Extract a reusable “post‑snapshot feature” helper**
   1. **add test first**
   - Move the Phase 5 notebook logic that builds
     `perf_agg`, `network_df`, and `team_df` into a small helper in
     `src/features/feature_factory.py` (or a new module, e.g.
     `src/features/post_split.py`) that:
     - accepts `experience`, `education`, `company_info`, and
       cut‑off mappings for founders (`person_id`) and companies
       (`company_id`),
     - uses `compute_founder_performance_aggregates`,
       `compute_network_features`, and `compute_team_size` to
       produce three tidy dataframes:
       `perf_agg[person_id, founder_*]`,
       `network_df[person_id, network_*]`,
       `team_df[company_id, team_size]`.
   - Keep the helper time‑respecting by using the same
     `cut_off_years` semantics already implemented in
     `feature_factory`.

3. **Extend the ranking feature pipeline to attach post‑split features**
   1. **add test first**
   - Update `FeaturePipeline._build_founder_features` (or wrap it
     with a new method) so that, after building the base founder
     rows, it:
     - derives an appropriate snapshot year per founder for ranking
       (e.g. from the company they are being scored on, such as
       `company_founded`),
     - builds the founder and company cut‑off mappings needed by the
       new helper,
     - merges `perf_agg` and `network_df` by `person_id`, and
       `team_df` by `company_id`, into the founder feature frame.
   - Ensure the final `FeatureMatrix.frame` includes all columns in
     `bundle.feature_columns` so that
     `select_columns(bundle.feature_columns)` succeeds.

4. **Align training and inference on the shared helper**
   - Refactor the Phase 5 training cells in
     `GV_Exercise_Iteration4.ipynb` to call the same helper used by
     ranking, so both training and inference compute the performance,
     network, and team features in one place.
   - Re‑run training once to confirm the saved `feature_columns` in
     `artifacts.json` still match the expected list.

5. **Add tests to guard against future drift**
   1. Add test cells inside ipynb to print and assert matrix shapes, columns etc.
