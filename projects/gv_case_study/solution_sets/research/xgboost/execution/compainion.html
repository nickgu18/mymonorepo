Final notebook structure (GV_Exercise_Simple.ipynb)

The exercise notebook becomes a thin orchestrator over the library functions above.

  1. Imports
    • from src.common.config import load_config
    • from src.data.loaders import load_raw, clean, load_targets
    • from src.models.training_pipeline import build_training_dataset
    • from src.data.splitters import split_by_industry
    • from src.features.pipeline import FeaturePipeline
    • from src.models.training import train_ranker
    • from src.models.io import load_model_bundle
    • from src.predict.inference import predict_batch
  2. Load config
    • Call cfg = load_config("configs/base.yaml").
  3. Load and clean data
    • Load curated training CSVs with raw_training = load_raw(cfg.data, dataset="training") and build clean_training = clean(raw_training).
    • Load ranking CSVs with raw_ranking = load_raw(cfg.data, dataset="ranking") and build clean_ranking = clean(raw_ranking).
    • Load the training targets with target_df = load_targets(cfg.data).
  4. Build training dataset
    • Call train_df = build_training_dataset(clean_training, clean_ranking, target_df, cfg.features) to get founder‑level rows (last founded company) with features, joined company_founded, corrected industry when applicable, and labels based on floor(log1p(multiple)) (with strict founder‑level leakage removal).
    • Optionally inspect label distribution / counts by industry.
  5. Split by industry
    • Select feature columns from train_df and the label column.
    • Call X_train, y_train, X_test, y_test, train_df_s, test_df_s, train_groups, test_groups, industry_names = split_by_industry(train_df, feature_columns, target_col="label", cutoff_year_map=cfg.data.cohort_cutoffs).
  6. Train ranker
    • Call ranker, metrics = train_ranker(X_train, y_train, X_test, y_test, params=cfg.model.hyperparameters, tracking_uri=cfg.tracking.uri, experiment_name=cfg.tracking.experiment_name, train_groups=train_groups, test_groups=test_groups, industry_names=industry_names).
    • Compute per‑industry SHAP importance: build a TreeExplainer on the trained ranker, compute SHAP on train/test features, aggregate by mean absolute value per cohort, and log/display top‑K features per industry.
    • Save ranker.json and artifacts.json using the existing IO utilities (or rely on MLflow artifacts if already wired).
  7. Run inference on ranking data
    • Reuse clean_ranking from the earlier load/clean step.
    • Create a FeaturePipeline and call build_matrix on clean_ranking to get the founder feature matrix for the interest list.
    • Load the trained bundle via load_model_bundle.
    • Score per industry: filter by industry, predict, and rank within each cohort. Use predict_batch to produce founder_id, industry, optional company_founded (when available), score, rank, and explanation (mandatory SHAP top‑K).
  8. Inspect results
    • Sort and display top founders per industry, plot simple diagnostics, etc.
