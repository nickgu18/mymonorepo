# ng-smash Skill

This skill provides an autonomous end-to-end workflow for resolving a Buganizer issue. It handles repository setup, implementation, PR creation, CI/CD validation, and communication.

## Prerequisites
- The agent must have access to the `gemini` CLI for Buganizer interactions.
- The agent must have access to the `gh` (GitHub) CLI for PR management.
- The project must be tracked by git.

## Workflow

### 1. Issue Triage
- Use the `gemini` CLI with the buganizer extension to read the details of the provided `issue_id`.
  - Command: `gemini -y -p "Read bug <issue_id> and summarize the requirements, acceptance criteria, and expected implementation."`
- Formulate an implementation plan based on the bug details.

### 2. Workspace Setup (Git Worktree)
- Do NOT modify the primary working directory immediately.
- Create a dedicated development environment using `git worktree` inside a `.factory` directory.
  - `mkdir -p .factory`
  - `git worktree add .factory/bug-<issue_id> main`
- Navigate to the new worktree: `cd .factory/bug-<issue_id>`
- Create and checkout a new feature branch: `git checkout -b feat/bug-<issue_id>`

### 3. Implementation & Validation
- Implement the changes according to the formulated plan.
- Ensure all project-specific formatting, linting, and testing standards are met locally.
- Use subagents or the `grep_search`/`glob` tools as needed to navigate the codebase safely.

### 4. Commit & PR Creation
- Stage the changes: `git add .`
- Write a clear, semantic commit message referencing the bug.
  - `git commit -m "fix: resolve b/<issue_id> - <brief description>"`
- Push the branch to the remote repository.
  - `git push -u origin feat/bug-<issue_id>`
- Create a Pull Request using the GitHub CLI.
  - `gh pr create --title "fix: resolve b/<issue_id>" --body "This PR implements the fix for b/<issue_id>. Automatically generated by ng-smash."`

### 5. CI/CD Iteration Loop
- Wait for GitHub Actions/CI checks to complete.
  - `gh pr checks --watch`
- **If checks fail:**
  - Retrieve the failure logs: `gh run view <run_id> --log`
  - Diagnose the failure, implement a fix, commit, and push.
  - Repeat the wait/check cycle until all CI checks pass.

### 6. Handoff & Cleanup
- Once the PR is green, leave a comment on the Buganizer issue to indicate handoff.
  - `gemini -y -p "Add comment to bug <issue_id>: Fix implemented and PR created successfully. CI checks are green. Please review: <PR_URL>"`
- Return to the original workspace root.
- Clean up the worktree.
  - `git worktree remove .factory/bug-<issue_id> --force`
